# AI 智慧学习系统 — 功能扩展任务清单

## 一、对话体验

### 1. Markdown 渲染 + 代码高亮
- **目标**：AI 回复支持 Markdown 格式渲染和代码语法高亮
- **技术方案**：前端安装 marked + highlight.js，ChatMessage 组件使用 v-html 渲染
- **验收标准**：标题、列表、加粗、代码块、语法高亮均正常显示

### 2. 会话重命名/删除
- **目标**：侧边栏支持会话重命名和删除操作
- **技术方案**：后端新增 PUT/DELETE /conversations/{id} 端点，前端侧边栏添加右键菜单或操作按钮
- **验收标准**：可重命名会话标题，可删除会话（含关联消息级联删除）

### 3. 消息复制/重新生成
- **目标**：消息气泡支持复制文本和重新生成 AI 回复
- **技术方案**：ChatMessage 组件添加操作按钮，复制用 navigator.clipboard，重新生成调用 /chat 端点
- **验收标准**：点击复制将消息文本写入剪贴板，点击重新生成覆盖当前 AI 回复

### 4. 对话搜索
- **目标**：侧边栏支持按关键词搜索会话
- **技术方案**：前端过滤 conversations 列表，或后端支持 search query 参数
- **验收标准**：输入关键词后实时过滤显示匹配的会话

---

## 二、用户功能

### 5. 用户设置（修改密码）
- **目标**：添加用户设置页面，支持修改密码
- **技术方案**：后端新增 PUT /auth/password 端点，前端新增 Settings.vue 页面
- **验收标准**：输入旧密码和新密码，验证通过后更新密码

### 6. 深色/浅色主题切换
- **目标**：支持深色和浅色两种主题切换
- **技术方案**：CSS 变量定义两套主题色板，Pinia store 管理主题状态，localStorage 持久化
- **验收标准**：点击切换按钮即时切换主题，刷新后保持选择

---

## 三、AI 能力

### 7. 多模型切换
- **目标**：支持切换不同的 MiMo 模型
- **技术方案**：前端添加模型选择器，请求时传递 model 参数，后端支持动态 model
- **验收标准**：下拉选择模型后，后续对话使用选定模型

### 8. 对话导出
- **目标**：支持将对话导出为 Markdown 文件
- **技术方案**：前端将 messages 格式化为 Markdown 文本，触发浏览器下载
- **验收标准**：点击导出按钮，下载包含完整对话记录的 .md 文件

## 四、系统增强

### 9. 移动端响应式优化
- **目标**：在手机端也能正常使用
- **技术方案**：CSS 媒体查询适配小屏幕，侧边栏改为抽屉式，输入区域固定底部
- **验收标准**：在 375px 宽度下布局正常，可正常对话

## 五、技术优化

### 11. 单元测试
- **目标**：为核心模块编写单元测试
- **技术方案**：后端用 pytest，前端用 Vitest
- **验收标准**：关键函数和 API 端点有测试覆盖

### 12. Docker 部署
- **目标**：支持 Docker 容器化部署
- **技术方案**：编写 Dockerfile（backend + frontend）和 docker-compose.yml
- **验收标准**：docker-compose up 一键启动完整系统

### 13. CI/CD 工作流
- **目标**：GitHub Actions 自动化检查
- **技术方案**：ci.yml 已创建，需 GitHub token 包含 workflow 权限后推送
- **验收标准**：push/PR 时自动运行后端检查和前端构建
