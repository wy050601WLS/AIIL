# AI 智慧学习系统 — 答辩文档

> 总时长：20 分钟

---

## 一、项目概述（2 分钟）

### 为什么做这个项目

学生学习有三个问题：
1. **问问题不方便** — 要等老师回复，或者网上搜半天
2. **学过的东西找不回来** — 之前问过的、学过的散落各处
3. **不知道自己学了多少** — 没有数据反馈

**AI 智慧学习**就是解决这三个问题：AI 随时回答问题，精华内容可以收藏成卡片，学习数据有面板可以看。

### 技术栈（简单说）

```
前端（用户看到的界面）：Vue3
后端（服务器逻辑）：Python FastAPI
数据库：MySQL
AI：MiMo 大模型
```

## 二、功能演示 + 实现讲解（10 分钟）

### 2.1 注册登录（1 分钟）

**演示**：注册一个账号 → 登录 → 改头像和昵称

**怎么实现的**：

注册的时候，密码不是直接存到数据库的，而是用 **bcrypt** 加密后再存。比如密码 `123456`，存到数据库变成一串乱码 `$2b$12$XyZ...`，这样即使数据库被偷了，别人也看不到原始密码。

登录的时候，后端验证密码是否正确，如果正确就生成一个 **JWT Token**（可以理解为一张「通行证」），返回给前端。前端把这张通行证存在浏览器里，以后每次请求都带上它，后端看到通行证就知道你是谁了。

```
用户输入密码 → 后端加密对比 → 正确 → 发通行证（Token）
                                    → 前端存起来 → 以后每次请求都带上
```

**关键代码**：
```python
# 后端 utils/security.py
def hash_password(password):
    return bcrypt.hashpw(password.encode(), bcrypt.gensalt())  # 加密

def verify_password(password, hashed):
    return bcrypt.checkpw(password.encode(), hashed)           # 验证

def create_token(user_id):
    return jwt.encode({"sub": str(user_id)}, SECRET_KEY)       # 生成通行证
```

### 2.2 AI 对话（3 分钟）— 最核心的功能

**演示**：发一个问题 → AI 一个字一个字蹦出来 → 代码有高亮 → 可以停止/重新生成

**怎么实现的 — 分三步**：

**第一步：前端发请求**

用户点「发送」，前端把消息发给后端。注意这里不是普通的 HTTP 请求，而是用 **SSE（Server-Sent Events）**，可以理解为「服务器单向推送」—— 后端每收到 AI 的一个字，就立刻推给前端，前端立刻显示。

```
用户点发送 → 前端发请求给后端 → 后端问 AI → AI 一个字一个字回答
                                    → 后端收到一个字就推给前端
                                    → 前端立刻显示
                                    → 所以用户看到的是「一个字一个字蹦出来」
```

**第二步：后端调 AI**

后端用 **httpx**（一个 HTTP 请求库）异步调用 AI 的接口。关键点是用 **stream（流式）** 模式，不是等 AI 全部回答完再返回，而是一边回答一边返回。

```python
# 后端 services/ai_service.py
async def stream_ai_api(history, model):
    async with httpx.AsyncClient() as client:
        async with client.stream("POST", ai_url, json=payload) as response:
            async for line in response.aiter_lines():
                # 每收到 AI 的一个字，立刻 yield 返回给前端
                content = parse_sse_line(line)
                if content:
                    yield content
```

**第三步：前端显示**

前端用 `fetch` 请求，通过 `ReadableStream` 读取数据流。每收到一个字，就追加到消息里，页面自动更新。

```javascript
// 前端 api/chat.js
const reader = response.body.getReader()
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  // 解析出 AI 回复的文字，调用 onToken 回调
  onToken(parsedText)  // 前端收到一个字，就显示一个字
}
```

**Markdown 渲染**：AI 回复可能包含标题、列表、代码块等格式。前端用 **marked** 库把 Markdown 转成 HTML，用 **highlight.js** 给代码加语法高亮，用 **DOMPurify** 过滤掉可能的恶意代码（防 XSS 攻击）。

**性能优化**：AI 回答很快，每秒钟可能几十个字。如果每个字都重新渲染一次 Markdown，页面会卡。所以加了 **150ms 防抖** —— 每 150 毫秒最多渲染一次，AI 停止回答后立刻渲染最终版本。

### 2.3 图片识别和语音输入（1 分钟）

**演示**：上传一张图片让 AI 看 → 用语音输入文字

**图片识别怎么实现的**：

用户上传图片后，前端用 `FileReader` 把图片转成 **base64**（一种把图片变成文字的方式）。发送消息时，文字和图片一起发给后端。

后端收到后，把图片转成 AI 能理解的格式（OpenAI 多模态格式），然后发给 AI。AI 模型如果是支持「看图」的（比如 GPT-4o），就能分析图片内容。

```
用户上传图片 → 前端转成 base64 文字 → 和文字一起发给后端
                                      → 后端转成 AI 能理解的格式
                                      → 发给 AI → AI 分析图片 → 返回结果
```

**语音输入**：用浏览器自带的 **Web Speech API**，对着麦克风说话，浏览器自动把语音转成文字，填到输入框里。支持中文，实时显示识别结果。

### 2.4 会话管理（1 分钟）

**演示**：新建对话、重命名、置顶、归档、搜索、导出

**怎么实现的**：

每个对话在数据库里是一条记录，有标题、是否置顶、是否归档等字段。

**搜索**是纯前端做的 —— 用户输入关键词，前端在已有的对话列表里过滤标题包含关键词的，不需要请求后端。

**置顶**就是把对话的 `pinned` 字段改成 `true`，显示的时候置顶的排前面。

**导出**是后端把对话的所有消息拼成 Markdown 文本，前端下载成 `.md` 文件。

**无限滚动**：长对话可能有几百条消息，一次全部加载会很慢。所以第一次只加载最近 50 条，用户往上滚动时再加载更早的消息。关键是要处理好滚动位置 —— 加载旧消息后，页面高度变了，滚动条会跳。解决方法是：加载前记住页面高度，加载后把滚动条调到原来的位置。

```javascript
// 前端 Chat.vue
if (滚动到顶部了) {
  const 旧高度 = 页面.scrollHeight
  await 加载更多消息()
  页面.scrollTop = 页面.scrollHeight - 旧高度  // 补偿滚动位置
}
```

### 2.5 知识卡片（0.5 分钟）

**演示**：从 AI 回复里提取卡片 → 标签筛选 → 编辑

**怎么实现的**：

AI 回复旁边有个「提取卡片」按钮，点击后把回复内容保存到数据库的 `knowledge_cards` 表里。每张卡片有内容、来源（来自哪个对话）、标签。

前端显示卡片列表时，可以用标签筛选，也可以搜索内容。编辑就是调用 `PUT /cards/{id}` 接口更新数据库。

### 2.6 学习资料和知识库（1.5 分钟）

**演示**：添加资料 → AI 搜索推荐 → 上传文档

**学习资料**：就是收藏的学习链接、笔记等。有标题、链接、分类、标签。前端用筛选框和搜索框过滤，纯前端操作。

**AI 搜索**：用户输入自然语言问题（比如「我有哪些编程资料？」），后端把用户的资料库信息发给 AI，让 AI 分析哪些资料相关，然后返回推荐结果和学习建议。

```
用户问「我有哪些编程资料？」
  → 后端加载用户的所有资料
  → 把资料列表发给 AI
  → AI 分析后返回推荐 + 理由 + 学习建议
  → 前端显示（Markdown 渲染）
```

**知识库文档**：用户上传 PDF/Word/TXT 文件，后端自动提取文字内容存到数据库。搜索时可以按标题和标签找文档。

文档解析用的库：
- PDF → **pypdf** 提取文字
- Word → **python-docx** 提取文字
- TXT/MD → 直接读取文件内容

### 2.7 学习面板（0.5 分钟）

**演示**：统计数据、趋势图、热门标签

**怎么实现的**：

后端写 SQL 查询统计：
- 对话总数：`SELECT COUNT(*) FROM conversations WHERE user_id = ?`
- 消息总数：`SELECT COUNT(*) FROM messages JOIN conversations ...`
- 最近 30 天每天的消息数：`SELECT DATE(created_at), COUNT(*) GROUP BY DATE`
- 热门标签：从知识卡片的 tags 字段拆分统计

前端的柱状图是用纯 CSS 做的 —— 每天一个 `<div>`，高度根据消息数计算，不需要图表库。

### 2.8 对话模板（0.5 分钟）

**演示**：打开模板面板 → 选一个模板 → 自动填充输入框

**怎么实现的**：

数据库里存了 23 个内置模板（翻译、解释、练习题等），用户也可以创建自己的模板。

输入框旁边有个按钮，点击弹出模板面板，按分类分组显示。点击模板就把内容填到输入框里。用户也可以把当前输入的内容保存为新模板。

启动时后端自动检查内置模板是否完整，缺失的自动补全。

## 三、技术架构（3 分钟）

### 3.1 整体结构

```
用户浏览器
    ↓ 打开网页
前端（Vue3）
    ↓ 发请求
后端（FastAPI）
    ↓ 查数据库        ↓ 调 AI
  MySQL            MiMo API
```

### 3.2 后端代码组织

```
backend/app/
├── main.py            → 入口文件，注册路由和中间件
├── config.py          → 配置文件，读取 .env 环境变量
├── database.py        → 数据库连接
├── models/            → 数据库表结构定义（7 张表）
├── schemas/           → 请求/响应数据格式校验
├── routers/           → API 接口定义（8 个模块，25+ 个接口）
├── services/          → 业务逻辑（AI 调用、文档解析等）
└── utils/             → 工具（JWT、密码加密）
```

**分工**：Router 接收请求 → Service 处理逻辑 → Model 操作数据库

### 3.3 数据库（7 张表）

```
用户表（users）
  ├── 对话表（conversations）── 消息表（messages）
  ├── 知识卡片表（knowledge_cards）
  ├── 学习资料表（learning_resources）
  ├── 知识库文档表（knowledge_documents）
  └── 对话模板表（prompt_templates）
```

一个用户有多个对话，一个对话有多条消息。删除用户时，所有关联数据自动删除（级联删除）。

### 3.4 前端代码组织

```
frontend/src/
├── views/         → 页面（登录、对话、卡片、面板、资料、设置）
├── components/    → 组件（侧边栏、消息气泡、输入框）
├── stores/        → 状态管理（用户、对话、卡片、模板等 8 个 store）
├── api/           → 请求封装（Axios）
├── router/        → 路由配置
└── style.css      → 全局样式变量
```

**状态管理**用 **Pinia**，每个功能模块一个 store，管理数据和操作方法。

## 四、技术亮点（3 分钟）

### 4.1 性能优化

| 问题 | 优化方法 | 效果 |
|------|----------|------|
| 打包太大（2MB） | Element Plus 按需导入 + highlight.js 只装需要的语言 | 缩小到 666KB |
| 流式输出卡 | Markdown 渲染加 150ms 防抖 | CPU 占用降 90% |
| AI 搜索阻塞 | 同步改异步 | 不卡其他请求 |
| 列表太慢 | 加索引 + 限制最大条数 | 数据多了也不卡 |

### 4.2 安全防护

| 风险 | 防护措施 |
|------|----------|
| 密码泄露 | bcrypt 加密存储 |
| 接口被刷 | 限流（60 次/分钟） |
| 恶意代码注入 | DOMPurify 过滤 AI 回复 |
| 越权操作 | 每个操作校验资源归属 |
| SQL 注入 | ORM 参数化查询 |

### 4.3 工程化

- **数据库迁移**：用 Alembic 管理数据库版本，改了表结构可以升级/回滚
- **容器化**：Docker Compose 一键部署（数据库 + 后端 + 前端）
- **CI 自动化**：GitHub Actions 每次提交自动跑测试和构建
- **单元测试**：23 个测试用例覆盖核心功能

### 4.4 解决的难点

| 难点 | 怎么解决的 |
|------|-----------|
| AI 回复慢，要等很久才开始显示 | 改成「边回答边显示」，不是等全部答完 |
| 流式输出时页面闪烁 | 加防抖，150ms 渲染一次，不是每个字都渲染 |
| 长对话几百条消息卡顿 | 只加载最近 50 条，滚动时再加载更多 |
| 重新生成失败会丢原来的回复 | 先备份旧回复，失败了自动恢复 |

## 五、总结（2 分钟）

### 项目成果

- **50+ 功能**：AI 对话、知识卡片、学习资料、知识库、学习面板、对话模板
- **33 次迭代**：从最初能聊天的基础版，到现在功能完整的系统
- **23 个测试**：核心功能都有自动化测试
- **体积优化 66%**：打包从 2MB 缩小到 666KB

### 不足之处

- 只支持一个 AI 服务商，不能自由切换
- 知识库只能提取文字，不能识别图片里的内容
- 没有协作功能，不能和同学共享资料

### 未来可以做的

| 学习计划 | 设定学习目标，追踪进度，提醒复习 |
