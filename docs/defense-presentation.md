# AI 智慧学习系统 — 答辩文档

> 总时长：20 分钟 | 结构：5 个章节

---

## 一、项目概述（2 分钟）

### 1.1 项目背景

传统学习有三个痛点：

| 痛点 | 表现 |
|------|------|
| 答疑效率低 | 遇到问题要等老师回复，或在海量搜索中筛选 |
| 知识碎片化 | 问答散落各处，难以回顾整理 |
| 缺乏数据反馈 | 不清楚学习频率、知识覆盖、薄弱环节 |

AI 对话可以即时解答，但通用工具（ChatGPT 等）缺少学习专属功能：知识沉淀、学习追踪、个性化偏好。

### 1.2 项目定位

**AI 智慧学习（AIIL）** — 面向学生的 AI 对话式学习助手，核心能力：

```
即时答疑 → 知识沉淀 → 学习可视化 → 多模态交互
```

### 1.3 技术栈

```
前端：Vue3 + Vite + Pinia + Element Plus
后端：Python FastAPI + SQLAlchemy
数据库：MySQL 8.0 + Alembic 迁移
AI：MiMo（OpenAI 兼容接口）
部署：Docker Compose + GitHub Actions CI
```

**选型理由**：FastAPI 原生支持异步和 SSE 流式响应，Vue3 组合式 API 开发效率高，MySQL 成熟可靠。

## 二、系统演示（8 分钟）

> 按功能模块逐一演示，每个模块 1-2 分钟

### 2.1 用户系统（1 分钟）

**演示要点**：
- 注册 → 登录 → JWT 自动鉴权
- 个人资料：emoji 头像、昵称
- 偏好设置：字体大小、消息密度、默认模型
- 深色/浅色主题切换

### 2.2 AI 对话（3 分钟）— 核心功能

**演示要点**：
1. **流式对话**：发送问题，观察逐字输出效果（SSE 真流式）
2. **Markdown 渲染**：AI 回复中的标题、列表、代码块、表格
3. **代码高亮**：编程问题的语法高亮 + 代码块复制按钮
4. **多模型切换**：下拉切换模型，选择带「👁 视觉」标签的模型
5. **图片识别**：上传一张图片（如题目截图），观察 AI 分析
6. **语音输入**：点击麦克风，中文语音转文字
7. **消息操作**：复制、编辑、删除、重新生成

**技术亮点**：
- SSE 真流式：后端收到 AI 每个 chunk 立即转发，非回放式
- 流式 Markdown 防抖：150ms 节流，避免每个 token 触发全量解析
- DOMPurify 防 XSS：AI 回复经安全过滤后渲染

### 2.3 会话管理（1 分钟）

**演示要点**：
- 新建会话（Ctrl+N）、搜索（Ctrl+K）

### 2.5 学习资料与知识库（1.5 分钟）

**演示要点**：

- **AI 搜索**：输入自然语言问题，AI 分析资料库并推荐

### 2.6 学习面板（0.5 分钟）

## 三、技术架构（3 分钟）

### 3.1 整体架构

```
┌─────────────────────────────────────┐
│          前端 (Vue3 + Vite)         │
│  Views → Components → Stores        │
│  API Layer (Axios + fetch SSE)      │
└──────────────┬──────────────────────┘
               │ HTTP + SSE
┌──────────────▼──────────────────────┐
│         后端 (FastAPI)              │
│  Routers → Services → Models        │
│  Pydantic Schemas + JWT Auth        │
└──────────┬───────────┬──────────────┘
           │           │
     ┌─────▼─────┐ ┌───▼──────────┐
     │ MySQL 8.0 │ │ MiMo AI API  │
     │ (Alembic) │ │ (OpenAI 兼容)│
     └───────────┘ └──────────────┘
```

### 3.2 后端分层

| 层级 | 职责 | 文件 |
|------|------|------|
| Router | 路由定义、请求校验、响应格式 | `routers/` (8 个模块) |
| Service | 业务逻辑、AI 调用、数据处理 | `services/` (4 个模块) |
| Model | ORM 映射、表结构定义 | `models/` (7 张表) |
| Schema | Pydantic 请求/响应模型 | `schemas/` (20+ 模型) |
| Utils | JWT、密码哈希、限流 | `utils/` + `limiter.py` |

### 3.3 数据库设计

```
users (1) ──→ (N) conversations (1) ──→ (N) messages
  │
  ├──→ (N) knowledge_cards
  ├──→ (N) learning_resources
  ├──→ (N) knowledge_documents
  └──→ (N) prompt_templates
```

**核心表**：7 张表，含级联删除、复合索引、JSON 字段（图片/偏好）。

### 3.4 核心实现模式

**SSE 流式对话**：
```
前端 fetch → ReadableStream → 逐 token 更新
后端 httpx.AsyncClient → async generator → StreamingResponse
```

**认证流程**：
```
登录 → JWT Token → localStorage
请求 → Authorization Header → HTTPBearer 校验
401 → 自动跳转登录页
```

## 四、技术亮点与难点（4 分钟）

### 4.1 性能优化（1.5 分钟）

| 优化项 | 措施 | 效果 |
|--------|------|------|
| 前端体积 | Element Plus 按需导入 + highlight.js 精简 | 1942KB → 666KB（-66%） |
| 流式渲染 | Markdown 解析 150ms 防抖 | CPU 占用降 ~90% |
| 后端异步 | resources/ask 改用 AsyncClient | 不阻塞事件循环 |
| 查询优化 | 复合索引 + 列表上限 + AI 上下文窗口 | 防止数据量增长后卡顿 |
| 懒加载 | 模板缓存、Tab 按需加载、profile 去重 | 减少冗余 API 调用 |

### 4.2 安全防护（1 分钟）

| 层面 | 措施 |
|------|------|
| 认证 | JWT + bcrypt 密码哈希 |
| 限流 | slowapi 全局 60/min，登录 10/min，注册 5/min，AI 20/min |
| XSS | DOMPurify 清理 Markdown 输出 |
| 归属校验 | 所有操作验证资源归属权 |
| CORS | 可配置跨域来源 |
| SQL 注入 | SQLAlchemy ORM 参数化查询 |

### 4.3 工程化实践（1 分钟）

- **数据库迁移**：Alembic 版本管理，支持升级/回滚
- **容器化**：Docker Compose 一键部署（MySQL + Backend + Frontend）
- **CI/CD**：GitHub Actions 自动运行 pytest + vite build
- **单元测试**：23 个测试用例，覆盖认证、对话、消息、功能
- **代码规范**：全量中文注释、模块文件头 docstring

### 4.4 遇到的难点与解决（0.5 分钟）

| 难点 | 解决方案 |
|------|----------|
| SSE 流式输出首 token 延迟高 | 从回放式改为真流式，收到 chunk 立即转发 |
| 流式输出时 Markdown 闪烁 | 150ms 防抖 + 加载结束立即渲染 |
| 长对话消息过多导致卡顿 | 上下文窗口 50 条 + 消息分页加载 |
| 重新生成失败丢失原回复 | 暂存旧消息副本，失败时自动恢复 |

## 五、总结与展望（1 分钟）

### 5.1 项目成果

- **50+ 功能项**：涵盖对话、卡片、资料、知识库、面板、模板
- **33 个迭代阶段**：从 MVP 到完整系统，持续迭代优化
- **23 个测试用例**：核心功能自动化测试
- **66% 体积优化**：前端构建体积从 1942KB 降至 666KB

### 5.2 已知不足

- AI 模型固定为单一服务商，未实现多供应商自动切换
- 知识库文档仅支持纯文本解析，不支持图片/表格提取
- 缺少协作功能（共享资料、团队学习）

### 5.3 未来方向

| 优先级 | 方向 |
|--------|------|
| 高 | RAG 检索增强：基于知识库文档的精准问答 |
| 高 | 对话分支：同一话题多角度探索 |
| 中 | 学习计划：设定目标、追踪进度、提醒复习 |
| 中 | 多语言支持：界面国际化 |
| 低 | 协作空间：共享资料和学习笔记 |
