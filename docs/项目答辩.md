# 项目答辩
- 如何借助于AI来梳理项目核心功能的落地:
    1. 让AI工具获取当前项目代码的上下文信息（让AI了解你的项目代码）
        - 进入到项目目录里面打开终端
        - `codegraph init -i`
        - 打开Claude code
        - `/init` --> 在项目根目录中创建一个`CLAUDE.md`（给Claude code用的一个项目的说明文件）
    2. 让AI工具梳理核心功能的实现逻辑
        - 提示词：请帮我梳理xxxx功能的实现逻辑，包括这个功能实现的核心技术栈及技术栈的相关介绍（是什么？为什么要用？如何使用的？）
    3. 让AI进行代码审查（功能逻辑合理性、性能与安全性、架构与可维护性）看是否功能实现合理，代码质量是否符合严格的标准（可选）
        - 提示词：请帮我审查现有实现该功能逻辑的代码，从代码的功能逻辑合理性、性能与安全性、架构与可维护性进行审查，看功能实现是否符合商业级或企业级产品的标准，并提出合理化的建议。
    4. 提示词：根据现有的功能实现逻辑（根据代码审查的结果 我准备从xxx方面进行优化）帮我整理功能实现笔记文档，方便后期复习使用。文档内容要包括：
        1. 此功能用到的核心技术栈有哪些，按照是什么、为什么要用、如何使用的这个思路进行整理。
        2. 功能的核心实现流程（画完整流程图）及相关代码文件、代码片段，还要有在实现的过程中遇到的核心问题/考虑到的问题及解决方案（把这个作为重难点）。
        3. 项目答辩话术。
        4. 后续要把这个功能写入到简历里面，应该如何进行编写。
        5. 如果把这个功能作为核心亮难点，面试话术怎么讲。
        6. 后续面试可能会追问的问题及答案。
